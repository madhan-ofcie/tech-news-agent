================================================================================
                    STEP-BY-STEP SETUP GUIDE
================================================================================

STEP 1: Create S3 Bucket
─────────────────────────────────────────────────────────────────────────────

Run this command in AWS CLI:

    aws s3 mb s3://your-tech-news-bucket

Replace "your-tech-news-bucket" with a unique name (bucket names must be
globally unique across all AWS accounts).


STEP 2: Verify Email in Amazon SES
─────────────────────────────────────────────────────────────────────────────

1. Go to AWS Console → Amazon SES
2. Select region: us-east-1 (has best sandbox access)
3. Click "Verified Identities"
4. Click "Create Identity"
5. Select "Email address"
6. Enter your email address
7. Click "Create"
8. Check your inbox for verification email from AWS
9. Click the verification link in the email

Note: In sandbox mode, you can only send to verified email addresses.


STEP 3: Create Lambda Function
─────────────────────────────────────────────────────────────────────────────

1. Go to AWS Lambda Dashboard
2. Click "Create Function"
3. Fill in details:
   - Function name: tech-news-agent
   - Runtime: Python 3.12
   - Architecture: x86_64
4. Click "Create Function"


STEP 4: Add Lambda Layer (requests library)
─────────────────────────────────────────────────────────────────────────────

Lambda doesn't come with the requests library by default, so we need to add it:

1. Go to AWS Lambda → Layers
2. Click "Create Layer"
3. Download requests library locally:

       pip install requests -t python/

4. Zip the python folder:

       zip -r requests-layer.zip python/

5. Upload to AWS Lambda Layer
6. Attach this layer to your tech-news-agent function


STEP 5: Set Environment Variables
─────────────────────────────────────────────────────────────────────────────

In Lambda function → Configuration → Environment Variables

Variable Name               Value
─────────────────────────────────────────────────────────────────────────────
GROQ_API_KEY              Your Groq API key from console.groq.com
SENDER_EMAIL              Your verified SES email address
RECIPIENT_EMAIL           Email where you want to receive daily news
S3_BUCKET                 Your S3 bucket name

IMPORTANT: Never put API keys in your code! Always use environment variables.


STEP 6: Configure Lambda Settings
─────────────────────────────────────────────────────────────────────────────

In Lambda function → Configuration → General Configuration:

  • Timeout: 2 minutes 30 seconds (150 seconds)
  • Memory: 256 MB
  • Ephemeral storage: 512 MB (default)

Why? Generating text and images takes time, and we need enough memory.


STEP 7: Deploy Lambda Code
─────────────────────────────────────────────────────────────────────────────

1. Copy the lambda_function.py code
2. Go to your Lambda function → Code tab
3. Paste the code into the editor
4. Click "Deploy"

The function is now ready!


STEP 8: Set Up EventBridge Scheduler
─────────────────────────────────────────────────────────────────────────────

1. Go to AWS Console → EventBridge → Schedules
2. Click "Create Schedule"
3. Fill in:
   - Name: tech-news-daily
   - Description: Daily tech news generation at 6 AM IST

4. Schedule expression: cron(30 0 * * ? *)
   
   Explanation:
   - 30 = 30 minutes
   - 0 = 0 hours (midnight)
   - * = every day
   - * = every month
   - ? = any day of week
   - * = every year
   
   This runs at 12:30 AM UTC = 6:00 AM IST

5. Select Target: AWS Lambda → tech-news-agent
6. Click "Create Schedule"

DONE! Your agent is now live! 🚀

================================================================================
                          SECURITY BEST PRACTICES
================================================================================

✓ API Keys Security
  • Store API keys in Lambda Environment Variables (encrypted at rest)
  • NEVER commit API keys to GitHub or any version control
  • Use AWS Secrets Manager for production deployments

✓ S3 Bucket Security
  • S3 bucket has private access only
  • Only Lambda function can read/write to it
  • Block public access enabled

✓ SES Restrictions
  • In sandbox mode initially (sends only to verified emails)
  • Can request production access once tested
  • All emails are logged in CloudWatch

✓ Code Security
  • No credentials hardcoded in source code
  • All sensitive data via environment variables
  • Lambda execution role has minimal permissions (principle of least privilege)

================================================================================
                         TROUBLESHOOTING GUIDE
================================================================================

Issue: "Lambda timeout exceeded"
Solution:
  • Increase timeout to 3 minutes in Lambda settings
  • Check if Groq API or Pollinations AI is responding slowly
  • Monitor CloudWatch logs for where it's hanging

Issue: "Permission denied uploading to S3"
Solution:
  • Verify Lambda execution role has S3 permissions
  • Check bucket name in environment variable
  • Ensure bucket exists and is in same region

Issue: "Email not received"
Solution:
  • Verify email address is verified in SES
  • Check SES is not in sandbox mode
  • Check spam/junk folder
  • Verify SENDER_EMAIL and RECIPIENT_EMAIL in env vars

Issue: "Groq API rate limit"
Solution:
  • Free tier has rate limits
  • Add delay between API calls
  • Consider upgrading Groq plan for production

Issue: "Invalid GROQ_API_KEY"
Solution:
  • Get new key from https://console.groq.com
  • Ensure no extra spaces in environment variable
  • Test key locally before deployment
